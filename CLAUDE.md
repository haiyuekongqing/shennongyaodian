# 中草药 Agent 开发约束指令 (必须严格遵守)

## 模块与架构原则
- 所有外部 API Key 必须通过 .env 文件读取，禁止硬编码在源码中。

## 医疗安全与防幻觉红线
- 任何返回给用户的医药建议接口，必须在数据结构中预留 disclaimer 字段。
- 所有的系统 Prompt 模板中必须包含：“如果你不知道答案，或者提供的参考资料中没有相关信息，请直接回答‘抱歉，我目前无法确认该信息’，严禁编造医学知识。”

## 数据库安全红线（必须遵守）
- 严禁在未获得用户明确许可的情况下清空向量数据库（Milvus）或图数据库（Neo4j）的数据。
- 如需清空集合（`clear_collection`）、删除节点（`clear_all`）、重置导入记录等破坏性操作，必须事先向用户说明原因并等待书面确认。
- 向量数据和图谱数据是系统的核心资产，误操作会导致所有知识丢失，必须视为生产线数据对待。
